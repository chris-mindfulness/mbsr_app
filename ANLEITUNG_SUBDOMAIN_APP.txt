ANLEITUNG: Subdomain app.mindfulpractice.de (Cloudflare + GitHub Pages)

Voraussetzung:
- Die Domain mindfulpractice.de gehoert dir und ist in Cloudflare verwaltet.
- Die App wird aktuell ueber GitHub Pages deployed.
- Registrar-Hinweis: Die Domain kann bei checkdomain.de registriert sein, wichtig ist nur,
  dass die Nameserver auf Cloudflare zeigen. DNS-Aenderungen erfolgen in Cloudflare.

1) DNS in Cloudflare anlegen
   - Record: CNAME
   - Name/Host: app
   - Target: chris-mindfulness.github.io
   - Proxy-Status: DNS only (graue Wolke), damit GitHub Pages TLS sauber funktioniert

   Optional (wenn du auch www.app.mindfulpractice.de willst):
   - Record: CNAME
   - Name/Host: www.app
   - Target: app.mindfulpractice.de

2) GitHub Pages Custom Domain setzen
   - Repo -> Settings -> Pages -> Custom domain
   - app.mindfulpractice.de eintragen
   - GitHub legt automatisch eine CNAME an (oder du legst sie selbst an)

   Optional: CNAME-Datei im Repo:
   - Datei: web/CNAME
   - Inhalt: app.mindfulpractice.de

3) Flutter Web Build auf Root setzen
   - Bei Custom Domain muss base-href "/" sein (nicht /mbsr_app/)
   - Beispiel:
     flutter build web --base-href "/" --release

   Wenn du GitHub Actions nutzt:
   - Workflow anpassen, damit base-href "/" gebaut wird

4) HTTPS aktivieren
   - Cloudflare: SSL/TLS -> Full (oder Full Strict, falls Zertifikat ok)
   - GitHub Pages: "Enforce HTTPS" aktivieren (falls verfuegbar)

5) Appwrite / CORS
   - In Appwrite eine Web-Plattform mit:
     https://app.mindfulpractice.de
     hinzufuegen

Hinweise:
- DNS-Aenderungen koennen 5-30 Minuten (manchmal bis 24h) brauchen.
- Wenn du die alte URL (github.io/mbsr_app) parallel behalten willst,
  brauchst du zwei getrennte Deployments wegen unterschiedlicher base-href.
